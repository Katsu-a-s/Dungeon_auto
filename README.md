# Dungeon Auto

Python + [pygame](https://www.pyga.me/) 製のローグライク風ダンジョン探索ゲーム。
メインファイルは **`dungeon_rev138.py`**(単一ファイル構成、約7,150行)。

> このリポジトリは Claude によるルーティン作業(1時間ごとの自動開発)で
> 少しずつ機能追加・改善が行われています。このREADMEも実装状況に合わせて
> 逐次更新されます。

## 遊び方

```bash
pip install pygame
python3 dungeon_rev138.py
```

- 画面サイズ 880x720、キーボード + マウス操作対応。
- セーブデータ等(`savefile*.json` / `autosave.json` / `settings.json` など)は
  実行時にリポジトリ直下へ自動生成されます(`.gitignore` 済みでコミット対象外)。

## 操作方法

ゲーム内タイトル画面右上の **`[F1] Help`** ボタン(または `F1` キー)から、
いつでもコントロール一覧を確認できます。

| 場面 | キー | 内容 |
|---|---|---|
| タイトル | 矢印キー / `Space` | メニュー選択 / ゲーム開始 |
| タイトル | `T` | 難易度切り替え(Easy/Normal/Hard) |
| タイトル | `N` | キャラクター選択 |
| タイトル | `G` | ゲームデータ(ロード/コンティニュー/設定) |
| タイトル | `R` | 記録(実績/統計/図鑑/エコーバトル/デイリーランキング) |
| タイトル | `Y` | デイリーチャレンジ開始 |
| タイトル | `H` | 隠しステージ(全クリア後に出現) |
| タイトル | `F1` | ヘルプ画面 |
| ダンジョン探索 | 矢印キー | 移動 |
| ダンジョン探索 | `Q` | セーブメニュー |
| ダンジョン探索 | `K` | スキルツリー |
| ダンジョン探索 | `I` | ポーション使用 |
| バトル | `A`/`P`/`B`/`R`/`D`/`F` | 攻撃/ポーション/爆炎石/逃走/防御/集中 |
| バトル | `Up`/`Down`, `Space`/`Enter` | コマンド選択/決定 |
| 共通 | `Esc` | 戻る・キャンセル |
| 共通 | `M` | 全音声(BGM/SE)をミュート/解除 |

## 現在の実装状況

### ダンジョン・進行
- 全 **3ステージ × 30階 = 90階**(`STAGE_LENGTH`/`STAGE_COUNT`)+ 全クリア後の隠しステージ(ボス戦)。
- 迷路生成 + フロア修飾(霧/豊穣/岩場/煌めき/静穏/幸運/共鳴/氷結/剛力/豊穣の森/静謐/呪い/静寂/快晴/精鋭の巣/豪奢/光輝/慈悲/護り/市場/実直/肥沃/揮発など、
  `FLOOR_MODIFIERS`、計28種類、NEW: Volatile Floor)。フロア特性は入室直後だけでなく
  探索中・バトル中も画面に小さく常時表示され、効果を忘れにくいように改善。
- **新フロア特性「Volatile Floor(揮発の床)」(NEW)**:このフロアでは、爆炎石
  (Blaze Gem)の与えるダメージが1.5倍(1000→1500)になる。従来のフロア特性は
  通常攻撃・クリティカル・被ダメージなどには関わっていたが、アイテムとして使う
  爆炎石そのものの威力に関わる特性が無かったため、新しい方向性として追加した。
- **新フロア特性「Fertile Floor(肥沃の床)」**:このフロアでは、探索中に
  見つかる食料アイテム(パン/肉/りんご)の回復量が1.5倍になる。従来のchilly/verdant
  Floorは食料の「消費速度」に関わる特性だったが、拾った食料そのものの「量」を
  増やす特性が無かったため、新しい方向性として追加した。
- **新フロア特性「Genuine Floor(実直の床)」**:このフロアでは、宝箱を
  開けてもミミックが混ざらなくなる。従来のフロア特性はダメージ・遭遇頻度・
  取引コストなどに関わるものばかりで、「宝箱そのものの安全性」に関わる特性が
  無かったため、新しい方向性として追加した。
- **新実績「Deep Delver」**:通算で(その1回のランで)フロア60に到達すると
  解除(称号: the Abyssal)。歴代最深到達フロア(`deepest_floor_reached`)は
  以前からプレイ統計に記録されていたが、それを目標にした実績が無かったため、
  Veteran(撃破数)などと同じ枠組みでやり込み目標として追加した。
- **探索率表示の色分け強化**:探索中HUDの「Explored: N%」表示が、
  従来は「ボーナス達成ラインを超えたかどうか」の白/緑の2色だけだったのに対し、
  さらに上の「Perfect」ライン(97%以上)に到達すると金色で表示されるように
  変更。階段に着くまで自分がどちらのボーナス階級を狙えているか分からなかった
  ため、探索中からリアルタイムでプレビューできるようにした。
- **新フロア特性「Bazaar Floor(市場の床)」**:このフロアでは、探索中に
  出会う旅の商人との取引コストが25%引きになる。従来のフロア特性はプレイヤーの
  戦闘・探索能力(攻撃力・被ダメージ・遭遇頻度など)に関わるものばかりで、
  アイテム交換の「コスト」自体に関わる経済寄りの特性が無かったため、
  新しい方向性として追加した。
- **新フロア特性「Warded Floor(護りの床)」**:このフロアでは呪いの床
  (踏むとSTR/DEFが下がるタイル)の効果を受けなくなる。既存のSerene Floorは
  毒を無効化する特性だったが、同じ「特定の状態異常をまるごと防ぐ」方向性を
  呪いの床に対しても持たせるため、防御寄りの新しい特性として追加した。
- **新フロア特性「Merciful Floor(慈悲の床)」**:このフロアでは罠(通常の罠・
  罠の宝箱)によるダメージが30%軽減される。既存のQuiet Floorは罠との「遭遇頻度」
  自体を下げる特性だったが、踏んでしまった罠の「痛さ」を直接和らげる特性が
  無かったため、防御寄りの新しい方向性として追加した。
- **新フロア特性「Radiant Floor(光輝の床)」**:このフロアではクリティカルヒットの
  倍率が通常のx2からx2.5に上がる。Fortunate Floorはクリティカルの「発生率」、
  Empowered Floorは通常攻撃の「威力」に関わる特性だったが、クリティカル自体の
  「威力」を直接伸ばす特性が無かったため、新しい方向性として追加した。
- **新フロア特性「Opulent Floor(豪奢の床)」**:このフロアでは宝箱の出現重みが
  約2倍になり、フロアあたりの宝箱の数が増える。従来のフロア特性はアイテムの
  「質」(bountiful/cursed)や遭遇頻度(veiled)には関わっていたが、宝箱そのものの
  「数」を直接増やす特性が無かったため、新しい方向性として追加した。
- ショートカット移動・キー長押しでの連続移動、氷床での自動スライドなど操作性の作り込み。
- 20種類以上の特別部屋(モンスターの巣・祠・裂け目・賭博場・キメラ部屋・血月イベント等)を
  フロアごとに抽選配置。

### バトル
- ターン制バトル(攻撃/ポーション/爆炎石/逃走/防御/集中)+ コンボ倍率システム。
- 毒/呪い/仲間バフ/トーテムバフなどの状態異常・バフ。
- 通常敵17種(フロア帯で出現テーブルが変化)+ エリート個体(出現率12%)。
- 10階ごとにボス出現、90階超の隠しステージにも専用ボス。
- クリティカルヒット(スキル「会心の一撃」で発生率上昇、Fortunate Floorでは
  スキル未取得でもそのフロア限定でクリティカルが発生するように、Radiant Floorでは
  倍率自体がx2→x2.5に上昇)。
- **「MASSIVE HIT!」演出**:一撃で500ダメージ以上を叩き出すと、既存の
  クリティカル/コンボフィニッシャー演出よりもさらに強調した金色フラッシュと
  大きめの画面シェイクが発生する。コンボ・クリティカル・Focus攻撃・Radiant Floor
  などを組み合わせて大ダメージを狙う「見せ場」がこれまで無かったため、
  一撃の破壊力そのものにも視覚的なご褒美を与えるように追加した。
- ボスはHPが50%を切ると「フェーズ2(激怒)」に移行し、攻撃力が上昇(HPバーが
  脈打つ赤に変化し `* ENRAGED *` 表示が戦闘終了まで持続し、激怒状態が一目で
  わかるように)。
- **コンボのダメージボーナス%表示**:バトル中の `COMBO x◯` 表示に、
  現在のダメージ倍率(例: `COMBO x3 (+30% dmg)`)を併記するように。従来は
  積んだコンボの「段数」しか見えず、実際にどれだけダメージが伸びているのか
  数値でわからなかったため、コンボを積む動機づけを明確にした。
- **逃走(`[R]un`)成功率のヒント表示**:コマンド選択で `Run` にカーソルを
  合わせると、隣に成功率の目安(`~60% success`)を表示するように。従来は
  成功率が非公開で、逃げるかどうかの判断材料が乏しかったため明確化。
- **新フロア特性「Tranquil Floor(静謐の床)」**:このフロアでは逃走成功率が
  +20%(表示される目安も連動して`~80%`などに変化)。逃走は選べても成功率が
  固定で運任せだったため、フロア特性としても関わりを持たせた。
- **新フロア特性「Cursed Floor(呪いの床)」**:このフロアでは敵の攻撃
  ダメージが+15%になる代わりに、宝箱などのアイテム出現率が+25%される
  ハイリスク・ハイリターン特性。従来のフロア特性はデメリットのみ/メリットのみが
  ほとんどで、「危険を承知で挑む」という選択肢が薄かったため、リスクとリターンを
  両方持つ特性として追加した。
- **新フロア特性「Veiled Floor(静寂の床)」**:このフロアではモンスターとの
  遭遇マスがフロア生成時に通常の約半分まで間引かれ、比較的静かに探索できる。
  これまでのフロア特性はステータス面の増減が中心で、そもそもの敵との遭遇頻度に
  関わるものが無かったため、探索そのものの体感が変わる新しい方向性の特性として
  追加した。
- **新フロア特性「Clear Floor(快晴の床)」**:このフロアに入ると、
  ミニマップが難易度設定に関係なく最初から全域表示される。従来「ミニマップ全開放」
  はEasy専用の一方的な優遇仕様だったが、フロア特性として当たれば
  Normal/Hardでも一時的に体験できるようにし、探索の見通しが良くなる
  「当たり」の特性を追加した。
- **新フロア特性「Elite Grounds(精鋭の巣床)」**:このフロアでは通常戦闘が
  エリート化する確率が12%上乗せされる(通常12% → 24%)。エリートは強い代わりに
  経験値ボーナスが付くため、危険を承知でハイリスク・ハイリターンの周回を狙える
  特性として追加した(既存のCursed Floorと同様、リスクとリターンを両方持つ方向性)。

### キャラクター・成長
- 4種の主人公タイプ(Warrior/Guardian/Scholar/Scout)でプレイスタイルを選択可能。
- レベル/経験値/所持品(ポーション・爆炎石・食料など)。
- 5系統×3段+奥義1つ、計16スキルのスキルツリー。
- 難易度3段階(Easy/Normal/Hard)でパラメータ調整可能。

### やり込み要素
- 実績システム(51種、「全フロア特性をすべて一度は経験すると解除される
  Floor Whisperer(称号: the Attuned)」を含む)+ 称号システム(実績達成で二つ名を獲得)。
- **新実績「Master Cartographer」(NEW)**:通算で50階分のフロアを完全探索
  (85%以上探索してから階段へ到達)すると解除(称号: the Master Cartographer)。
  フロアの完全探索回数(`floors_fully_explored`)は既存の「Explorer」実績
  (10階)のために記録されていたが、より長くやり込むプレイヤー向けの
  上位目標が無かったため、同じ記録を活かした上位版として追加した。
- **新実績「Chain Reaction」(NEW)**:バトルで通算25回コンボフィニッシャーを
  発動すると解除(称号: the Chainmaster)。コンボフィニッシャー自体は初回発動で
  解除される「Combo Finisher」実績としてすでに評価されていたが、Alchemist/
  Demolitionistなど他の消費・行動系実績と同様、繰り返し発動し続けることを
  評価する累積目標が無かったため、既存の記録(`combo_finishers_used`)を
  そのまま活かせるやり込み目標として追加した(実績一覧にも現在値/しきい値の
  進捗が表示される)。
- **新実績「Combo King」**:バトル中に20連続コンボを達成すると解除
  (称号: the Combo King)。歴代最高コンボ数(`highest_combo_reached`)は
  以前からプレイ統計に記録されていたが、それを目標にした実績が無かったため、
  既存の記録をそのまま活かせるやり込み目標として追加した(実績一覧にも
  現在値/しきい値の進捗が表示される)。
- **新実績「Fortified」**:バトルで防御の薬(Defense Pill)を通算25個
  使用すると解除(称号: the Fortified)。消費アイテムのうちポーション
  (Alchemist)と爆炎石(Demolitionist)にはすでに専用実績があったが、
  もう一つの消費アイテムである防御の薬だけ記録も実績も無かったため、
  同じ枠組みで抜けを埋めるやり込み目標として追加した(プレイ統計にも
  「Defense Pills used in battle」として表示)。
- **新実績「Treasure Hunter」**:通算150個の宝箱を開けると解除
  (称号: the Treasure Hunter)。宝箱を開けた回数自体はプレイ統計
  「Treasure chests opened」として以前から記録されていたが、それを目標にした
  実績が無かったため、Veteran(撃破数)と同じ枠組みでやり込み目標として追加した。
- **実績一覧に進捗表示**:記録画面の実績一覧で、まだ解除していない
  累積系の実績(Alchemist/Marathoner/Executioner/Veteran/Treasure Hunter/
  Fortifiedなど)の横に「現在値/しきい値」(例: `(320/500)`)を表示するように。
  従来はTraps triggeredのみ個別に進捗が見えていたが、他の多くの実績はあと
  どれだけ頑張ればよいか分からなかったため、まとめて確認できるように改善した。
- **新実績「Sharpshooter」**:バトルの一撃で500ダメージ以上を与えると解除
  (称号: the Sharpshooter)。コンボ・クリティカル・Focus攻撃を組み合わせて
  一撃の火力を伸ばす遊び方自体はできたが、それを直接評価する実績が無かったため、
  「MASSIVE HIT!」演出と連動する新しいやり込み目標として追加した(プレイ統計にも
  歴代最高の一撃ダメージ「Hardest single hit dealt」として表示)。
- **新実績「Marathoner」**:通算で10,000歩歩くと解除(称号: the Wanderer)。
  歩数は毎歩ディスクへ書き込むと負荷が大きいため、プレイ時間の蓄積と同じ方式で
  メモリに貯めてから既存のセーブ/階段移動などのタイミングでまとめて記録するように
  実装(プレイ統計画面にも「Steps taken」として表示)。
- **新実績「Executioner」**:バトルで通算100,000ダメージを与えると解除
  (称号: the Executioner)。攻撃がヒットするたびにダメージ量をプレイ統計
  「Total damage dealt」として積算するようにし、コンボやクリティカルヒットなど
  瞬間的な派手さはあっても記録に残っていなかった「与えたダメージの総量」を
  長期的なやり込み目標として追加した。
- **タイトル画面の実績進捗表示**:タイトル画面の `[R] Records` ボタン横に
  「解除数/全実績数」(例: `12/40`)を常時表示。従来は記録画面を開かないと
  進捗がわからなかったため、タイトル画面だけで進み具合を確認できるように改善。
- 図鑑(Bestiary):遭遇したモンスター/ボス/発見アイテムを記録・詳細表示。
- プレイ統計画面、セーブ3スロット+オートセーブ、デイリーチャレンジ(共通シード+ランキング)。
  統計には通算撃破数・宝箱開封数・通算で到達した最深フロア・通算の逃走回数・
  通算のポーション使用回数・歴代最高コンボ数に加え、
  **通算で与えたダメージ量(Total damage dealt)**も記録。
- **エコーバトル**:これまで倒したボスをフロア別に再挑戦できるモード。
- **新実績「Demolitionist」**:バトルで爆炎石(Blaze Gem)を通算30個使用すると
  解除(称号: the Demolitionist)。ポーション使用回数は「Alchemist」実績として
  既に記録されていたが、もう一つの消費アイテムである爆炎石には記録も実績も
  無かったため、同じ枠組みで攻めの選択肢にもやり込み目標を追加した。
  プレイ統計にも「Blaze Gems used in battle」として表示。
- **新実績「Veteran」**:バトルで通算500体の敵を倒すと解除
  (称号: the Veteran)。撃破数(`total_kills`)はプレイ統計にずっと記録されて
  いたものの、それ自体を目標にした実績が無かったため、既存の記録をそのまま
  活かせるやり込み目標として追加した。

### UI/UX
- **ボスHPバーのフェーズ2ラインの目印(NEW)**:ボス戦・エコーバトルでは、
  HPバーの50%位置(フェーズ2/激怒に切り替わるライン)に薄い縦線を常時表示
  するように。従来はHPが実際に50%を切った瞬間に初めて「ENRAGED」だと
  わかったが、戦闘開始時点からあとどれだけ削ればフェーズ2に入るのか目視で
  見通せなかったため、事前に狙いを定められるように追加した。
- タイトル画面はマウスクリック操作にも対応した階層メニュー(つやのあるボタンUI)。
- BGM/SE音量を個別調整できる設定画面 + `M`キーでのワンタッチ全ミュート切り替え。
- **画面シェイクのON/OFF設定**:設定画面(`[O] Settings`)に「Screen Shake」
  トグルを追加。被弾・クリティカル・コンボフィニッシャー時の画面シェイク演出は
  臨場感を高める一方、酔いやすい・目が疲れやすいプレイヤーには不快な場合が
  あったため、いつでもオフにできるアクセシビリティ設定として追加した(設定は
  `settings.json` に永続化)。
- **画面フラッシュのON/OFF設定**:設定画面に「Screen Flash」トグルを
  追加。クリティカルヒット・コンボフィニッシャー・自己ベストコンボ更新・
  希少アイテム発見の際に画面全体が光るフラッシュ演出は爽快感を高める一方、
  光過敏なプレイヤーには負担になりうるため、Screen Shakeと同様にいつでも
  オフにできるアクセシビリティ設定として追加した(設定は`settings.json`に
  永続化)。
- 実績・統計一覧のスクロール対応、図鑑詳細のクリックプレビュー。
- **ヘルプ画面(`[F1]`)**:操作方法をタイトル/探索/バトル別にまとめて確認可能。
- **低HP警告**:残りHPが最大値の20%を切ると、探索中・バトル中を問わず
  画面端が心拍のように速く脈打つ赤い縁取りで警告(崩落演出とは脈拍の速さで区別)。
- **食料残量の事前警告**:従来は食料が完全に0になった瞬間だけ数値が
  赤く点滅していたため、飢餓ダメージを受けてから慌てて食料を探すことが
  あった。残り30以下になった時点でオレンジ色に点滅するようになり、0になる
  前に余裕を持って食料(ポーション交換や落ちている食料アイテム等)を
  確保できるように改善。
- **ポーション切れの表示強調**:画面左下のステータスパネルで、所持
  ポーション数が0になると、他の数値と同じ白色ではなく灰色に変わるように
  なった。従来は「0」という数字も他の数値と見た目が同じで見落としやすかった
  ため、パッと見で「もう回復手段が無い」ことに気づけるようにした。
- **爆炎石切れの表示強調**:同じステータスパネルで、爆炎石の所持数が
  0になった場合もポーションと同様に灰色表示になるように変更。従来は
  ポーションのみ灰色化されており、もう一方の消費アイテムである爆炎石は
  0になっても他の数値と見分けが付かなかったため、表示ルールを揃えて改善した。
- **ポーション/爆炎石の残り僅か事前警告(NEW)**:画面左下のステータス
  パネルで、ポーション・爆炎石の残り数が1個になった時点で、食料の事前警告
  (残り30以下でオレンジ点滅)と同じ考え方でオレンジ色に点滅するように
  なった。従来は0個になって灰色化するまで数値の見た目に変化が無く、
  「あと1回で尽きる」という状況に気づきにくかったため、0になる前に
  ワンテンポ早く気づけるように改善した。
- **クリティカル/コンボフィニッシャー演出**:クリティカルヒットやコンボ
  フィニッシャーが発動した瞬間、画面全体が短く光るフラッシュ演出を追加し、
  攻撃の爽快感を強化(フィニッシャーはより長く・ピンクがかった色で発光)。
- **ボス階警告**:未撃破のボスがいるフロアに足を踏み入れると、
  「Welcome to floor」表示と一緒に脈打つ赤文字で警告メッセージが表示され、
  階段に着く前にポーションや爆炎石の準備をする猶予が生まれるように。
- **ボス階までの残り階数表示**:ボス階まで残り3階以内になると、
  探索中のHUDに「Boss floor in N」と表示されるように。従来はボス階に
  足を踏み入れた瞬間にしか警告が無く、直前まで準備するタイミングが
  分からなかったため、少し早めに心構えできるように追加した。
- **画面シェイク演出**:被弾時・クリティカルヒット・コンボフィニッシャー発動時に
  画面が数フレームだけ小刻みに揺れるようになり、打撃の重みを強化(ボスの
  フェーズ2/激怒中に被弾すると揺れが一段大きくなる)。
- **ダメージポップアップ**:攻撃がヒットした瞬間、命中した対象(敵/プレイヤー)の
  頭上にダメージ数値がふわっと浮かび上がってフェードアウトするように。固定の
  メッセージ欄だけでは伝わりにくかった「どこで何が起きたか」を視覚的に補強し、
  クリティカルヒットは金色・コンボフィニッシャーはピンク色で一回り大きく強調表示。
- **実績解除トースト**:実績を解除した瞬間、画面上部にバッジ画像付きの
  ゴールドのバナーが上からスライドインして「ACHIEVEMENT UNLOCKED!」と実績名を
  表示するように。従来は他の雑多な通知と同じ地味な1行テキストで、探索中/バトル中
  問わず気づきにくかったが、専用の演出にしたことで達成感がひと目で伝わるように
  なった(探索・バトル・メニューなどどの画面でも表示される)。
- **自己ベストコンボ更新演出**:バトル中に歴代最高コンボ数
  (プレイ統計の「Highest combo streak reached」)を更新した瞬間、
  「New best combo! x◯」というメッセージと共に、クリティカルヒットと同じ
  画面フラッシュ機構を金色で流用して光らせるように。従来は記録自体は
  裏で更新されていても画面上には何の合図も無く、コンボを伸ばす動機づけが
  弱かったため、更新の瞬間にひと目で気づけるように追加した(1バトルにつき
  演出は一度だけに絞り、連続ヒットのたびに出て煩わしくならないようにした)。
- **希少アイテム発見演出**:宝箱から指輪・アミュレットなど希少な
  アイテム(tier 5〜8)が出た瞬間、クリティカルヒットと同じ画面フラッシュ
  機構を金色で流用して光らせ、レベルアップと同じジングルを鳴らすように。
  従来はポーションや爆炎石と同じ地味なアイテム表示だけで、レアなアイテムを
  引いた瞬間の特別感が薄かったため、視覚・聴覚の両方で「当たり」を
  はっきり伝えるようにした。

### サウンド・アセット
- `image/`:敵・ボス・アイテム・床・エフェクトなど201点のスプライト。
- `sound/`:ステージ別BGM(タイトル/フィールド/バトル、全ステージ分)+ SE + ジングル(レベルアップ/勝利/ゲームオーバー)。
- **実績解除ジングル**:実績解除トーストが表示される瞬間、レベルアップ
  ジングルを再生して音でも達成に気づけるように(従来はゴールドバナーの
  スライドインのみで、BGM/SEに気を取られていると見逃すことがあった)。
- **希少アイテム発見ジングル**:宝箱から指輪・アミュレットなど希少な
  アイテムが出た瞬間もレベルアップジングルを再生するように(実績解除ジングルと
  同じ「再生待ちフラグ」の仕組みを流用。ミュート設定・SE音量設定にも追従する)。
  ミュート設定・SE音量設定にも追従する。

## 開発方針

- 既存のコードスタイル・命名規則を踏襲。
- 1回の自動実行あたりの変更は小〜中規模に留め、既存のゲームプレイを壊さない。
- 変更のたびに構文チェック(`py_compile`)・ヘッドレス動作確認
  (`SDL_VIDEODRIVER=dummy` での起動テスト)を実施。
- メインファイルは直接書き換えず、`cp` で複製してから編集する。
  `dungeon_revNNN.py` のNNNを1つ上げた新しいファイル名で保存し
  (例: `dungeon_rev123.py` → `dungeon_rev124.py`)、古いファイルは
  変更履歴としてそのまま残す。**実行・プレイ対象は常に番号が最大の
  ファイル。**バージョンごとの変更履歴をファイル名からも追えるようにする。

## 自動実行環境についてのメモ(つまずいた点)

- **新しい敵モンスターを追加する場合は先にスプライト素材の有無を確認すること**:
  通常敵0〜16種+ボス専用画像+隠しボス画像で `image/enemy0.png`〜`enemy20.png`
  (一部は`enemy_crystal_slime.png`等の個別ファイル名)がすべて使用済みで、
  空いている敵画像スロットが無い。画像生成手段が無い実行環境で新モンスターを
  実装しようとすると素材不足で頓挫するため、今回はフロア特性・実績・UI改善の
  3点に絞って実装した。新モンスターを追加したい場合は、まず新しいスプライト
  画像を`image/`に用意できるかどうかを最初に確認すること。
- 自動開発を行っている実行環境には `gh` CLI が入っておらず、GitHub操作
  (PR作成・確認・マージ等)は GitHub MCP サーバー経由のツールで行っている。
  もし別環境で `gh` コマンドが使えない場合は、PATH の通った `gh` の
  フルパスを確認するか、GitHub MCP/API 経由の代替手段に切り替えること。
- テスト用の実行環境には `pygame` がプリインストールされていないことがある。
  `py_compile` は問題なく通っても、ヘッドレス起動テスト
  (`SDL_VIDEODRIVER=dummy`)の前に `pip install pygame` が必要になる場合がある。
- ヘッドレス動作確認で `runpy.run_path()` を使うと戻り値の名前空間が
  実際にロードされたモジュールの `__globals__` とは別物(コピー)になり、
  グローバル変数を書き換えても関数側の挙動に反映されない。フロア特性など
  グローバル状態に依存するロジックを対話的にテストする場合は
  `importlib.util.spec_from_file_location` + `module_from_spec` +
  `exec_module` で実モジュールとしてロードすること。
- 効果音配列 `se`(`pygame.mixer.Sound` のリスト)は `main()` 関数内で
  ローカル変数として作られており、`global se` 宣言もされていないため、
  `main()` の外で定義された関数(`move_player()` など)から直接
  `se[n].play()` を呼ぶと `NameError` になる(実際にこのPRで一度踏んだ)。
  `main()` の外から効果音を鳴らしたい場合は、既存の
  `achievement_sound_pending` と同じ「フラグを立てて `main()` の
  メインループ側で再生する」パターンに倣うこと。
- `importlib` で実モジュールとしてロードする際、モジュール冒頭で
  `pygame.image.load(...).convert_alpha()` を呼んでいる箇所があるため、
  `pygame.init()` と `pygame.display.set_mode((880, 720))` を
  **`exec_module()` より前に**実行しておかないと
  `pygame.error: video system not initialized` で読み込みに失敗する。
- `main()` は `while True` の無限ループなので、ヘッドレスで動作確認する際は
  `pygame.event.get` を数十フレーム分だけ空リストを返し、その後
  `pygame.QUIT` イベントを1件返すダミー関数に差し替えると、
  `sys.exit()` 経由でクリーンに抜けられる(戻り値は`SystemExit`として
  捕捉する)。
- このPRのテスト中、`draw_crit_flash()` の透明度計算が
  `crit_flash_timer > CRIT_FLASH_FRAMES` の場合(コンボフィニッシャー
  `+4`・自己ベストコンボ更新`+2`)に255を超えてしまい、
  `pygame.Surface.fill()` が `ValueError: invalid color argument` で
  クラッシュする不具合を発見した(rev133時点から存在していた既存バグ)。
  0〜255の範囲を超えうるアルファ値/色成分は、ダメージポップアップの
  実装と同様に必ず `max(0, min(255, ...))` でクランプすること。
  「起動するか」だけでなく、実際にトリガー値を再現して関数を呼ぶ
  ピンポイントなテストをしないと見つからない類のバグなので、
  新しい演出を追加・変更したときは可能な範囲で実際の値を使った
  再現テストも行うとよい。
